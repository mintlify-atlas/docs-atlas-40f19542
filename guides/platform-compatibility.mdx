Not all speech recognition features are available on every platform or OS version. Before using a specific capability, check the tables below to confirm it is supported in your target environment. Features marked with `*` have additional conditions noted in the footnotes.

<Note>
  Compatibility data as of July 2025.
</Note>

### Notes by feature

- **Continuous recognition** тАФ Android 12 and below do not support the `continuous` option. On those devices it is ignored.
- **On-device recognition** тАФ requires an offline language model to be downloaded on Android 13+. See [On-device recognition](/guides/on-device-recognition).
- **Audio recording** тАФ saves microphone input to a local WAV file. Requires Android 13+. See [Audio recording](/guides/audio-recording).
- **Audio file transcription** тАФ transcribes a local file instead of the microphone. Requires Android 13+. See [File transcription](/guides/file-transcription).
- **Voice processing** (`iosVoiceProcessingEnabled`) тАФ iOS only. Adds signal processing to reduce microphone feedback from speakers. May lower speaker playback volume.
- **Contextual strings** тАФ biases the recognizer toward custom vocabulary. Works on both platforms but performs more reliably on iOS.
- **Punctuation** тАФ on Android 13+, only available with on-device recognition. Controlled by `addsPunctuation` (maps to `EXTRA_ENABLE_FORMATTING` on Android).
- **Language detection** тАФ Android 14+ only, requires the `com.google.android.as` service and `EXTRA_ENABLE_LANGUAGE_DETECTION`. See [Language detection](/guides/language-detection).
- **Word confidence & timing** тАФ per-word timestamps and confidence scores from the `segments` field in the `result` event. On Android, only available on Android 14+ with on-device recognition.
- **Offensive word masking** тАФ Android 13+ only. Controlled by `androidIntentOptions.EXTRA_MASK_OFFENSIVE_WORDS`.

### Notes by platform

- **Android (React Native)** тАФ continuous mode and audio recording are only available on Android 13+. The recognition engine can be changed via `androidRecognitionServicePackage`.
- **iOS (React Native)** тАФ uses Siri and `SFSpeechRecognizer` natively.
- **Chrome Desktop** тАФ implemented via the `webkitSpeechRecognition` prefix. Uses Google's server-based recognition.
- **Safari Desktop тЙе v16** тАФ implemented via the `webkitSpeechRecognition` prefix. Requires Siri to be enabled in macOS system settings.
- **Chrome on Android** тАФ supported, but results handling differs slightly from Chrome Desktop.
- **Chrome on iOS** тАФ not working as of the last test in 2023.
- **Edge on Windows** тАФ `SpeechRecognition` API is present but requires the Azure speech component. Verify it is installed at `edge://components`.
- **Edge on Mac ARM** тАФ `SpeechRecognition` API is present but the Azure speech component is not available.
- **Brave Desktop** тАФ as of July 2025, Brave is waiting for improvements to the Web Speech API for on-device recognition and does not ship a server-based fallback.
- **Firefox Desktop** тАФ no `SpeechRecognition` implementation.

<Tip>
  When targeting the web, use `isRecognitionAvailable()` at runtime to check whether the current browser has a working `SpeechRecognition` implementation. Note that some browsers (such as Brave) expose the API object but do not implement it, so runtime detection alone may not be sufficient тАФ consider cross-referencing with a user-agent check for those edge cases.
</Tip>
